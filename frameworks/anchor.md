# Anchor Framework Setup Rules

When integrating an Anchor program, securely and efficiently initialize the Anchor `Program` instance within a React Context, making it globally available.

## 1. Dependency Verification
Ensure these dependencies are in `package.json`:
```bash
npm install @coral-xyz/anchor @solana/wallet-adapter-react @solana/web3.js
```

## 2. v1 Isolation Warning
> **CRITICAL:** `@coral-xyz/anchor` v0.31.x depends on `@solana/web3.js` v1 internally. All v1 types (`PublicKey`, `Transaction`, `Keypair`) MUST remain confined to the Anchor provider layer. Never export or pass v1 types into hooks, components, or any `@solana/kit` code. Convert at boundaries:
> ```typescript
> import { address } from '@solana/kit';
> const v2Address = address(anchorPublicKey.toBase58());
> ```

## 3. Program Provider Architecture
Architect a dedicated custom hook that initializes the Anchor `Program`.

### Requirements:
1. **Wallet & Connection:** Consume `useConnection` and `useWallet` from `@solana/wallet-adapter-react`.
2. **Memoization:** The `Program` instance MUST be wrapped in `useMemo`. The dependency array MUST be `[connection, wallet.publicKey]` — using `[connection, wallet]` causes infinite re-renders because the `wallet` object reference changes on every render.
3. **Null Safety:** Return `null` if the wallet is disconnected. Never throw during rendering.

### Production-Ready Example:
```typescript
import { Program, AnchorProvider } from "@coral-xyz/anchor";
import { useConnection, useWallet } from "@solana/wallet-adapter-react";
import { useMemo } from "react";
import type { MyProgram } from "../types/my_program";
import idl from "../idl/my_program.json";

export function useMyProgram() {
  const { connection } = useConnection();
  const wallet = useWallet();

  const program = useMemo(() => {
    if (!wallet.publicKey) return null;

    const provider = new AnchorProvider(
      connection,
      wallet as any,
      { commitment: 'confirmed', preflightCommitment: 'confirmed' }
    );

    return new Program<MyProgram>(idl as MyProgram, provider);
  }, [connection, wallet.publicKey]);

  return program;
}
```

## 4. Transaction Confirmation
Never use the deprecated `provider.sendAndConfirm()` without a proper confirmation strategy. Use blockhash-based confirmation:
```typescript
const latestBlockhash = await connection.getLatestBlockhash('confirmed');
const signature = await program.methods
  .myInstruction(args)
  .accounts({ /* ... */ })
  .rpc();

await connection.confirmTransaction(
  { signature, ...latestBlockhash },
  'confirmed'
);
```

## 5. Anchor Error Parsing
Anchor programs return structured errors with codes and messages. Always decode them for the user:
```typescript
import { AnchorError } from "@coral-xyz/anchor";

try {
  await program.methods.myInstruction(args).accounts({}).rpc();
} catch (err) {
  if (err instanceof AnchorError) {
    // err.error.errorCode.code — e.g., "InsufficientFunds"
    // err.error.errorCode.number — e.g., 6001
    // err.error.errorMessage — human-readable message
    console.error(`Program error ${err.error.errorCode.number}: ${err.error.errorMessage}`);
  } else {
    // RPC or network error — inspect err.logs if available
    throw err;
  }
}
```

## 6. IDL Types
**CRITICAL:** Never cast the IDL to `any`. Always use the TypeScript types generated by `anchor build`, located in `target/types/`. Import the generated interface and pass it as a generic to `Program<T>`. This ensures full IntelliSense and type-safety across all downstream hooks. If the types file is missing, run `anchor build` first.

Once the provider is architected, proceed to [generators/hooks.md](../generators/hooks.md) to build data-fetching and mutation layers.
