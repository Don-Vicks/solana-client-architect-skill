# Anchor Framework Setup Rules

When tasked with integrating an Anchor program, your goal is to securely and efficiently initialize the Anchor `Program` instance within a React Context, making it globally available to the application.

## 1. Dependency Verification
Ensure the following dependencies are present in `package.json`. If missing, generate the install command for the user:
```bash
npm install @coral-xyz/anchor @solana/wallet-adapter-react @solana/web3.js
```
*Note: `@coral-xyz/anchor` currently relies internally on `@solana/web3.js` version 1. You must use it for Anchor-specific setups, but keep its legacy types isolated from the rest of your modern UI logic wherever possible.*

## 2. Program Provider Architecture
You must architect a dedicated React Context or a robust custom hook that initializes the Anchor `Program`.

### Implementation Requirements:
1. **Wallet & Connection:** Consume `useConnection` and `useWallet` from `@solana/wallet-adapter-react`.
2. **Memoization:** The `Program` instance MUST be wrapped in a `useMemo` block. Re-instantiating the `Program` on every render will cause catastrophic performance degradation and infinite re-renders in downstream React Query hooks.
3. **Null Safety:** The hook must gracefully return `null` or `undefined` if the wallet is disconnected. It should never throw a fatal error during rendering.

### Production-Ready Example Snippet:
```typescript
import { Program, AnchorProvider } from "@coral-xyz/anchor";
import { useConnection, useWallet } from "@solana/wallet-adapter-react";
import { useMemo } from "react";
import type { MyProgramType } from "../types/my_program"; // Always use the generated types!
import idl from "../idl/my_program.json";

export function useMyProgram() {
  const { connection } = useConnection();
  const wallet = useWallet();

  const program = useMemo(() => {
    // Graceful degradation if wallet is not connected
    if (!wallet || !wallet.publicKey) return null;
    
    // AnchorProvider requires a wallet interface that matches its internal NodeWallet.
    // The standard wallet-adapter wallet object satisfies this at runtime.
    const provider = new AnchorProvider(
      connection, 
      wallet as any, 
      { commitment: 'confirmed', preflightCommitment: 'confirmed' }
    );
    
    // Strongly type the program instance
    return new Program<MyProgramType>(idl as MyProgramType, provider);
  }, [connection, wallet]);

  return program;
}
```

## 3. IDL Types
**CRITICAL:** Never cast the `idl` to `any`. Always use the TypeScript types generated by Anchor (`anchor build`) located in `target/types/`. Import the generated interface and pass it as a generic to the `Program` instantiation (e.g., `Program<MyProgram>`). This ensures that all downstream hooks benefit from full intellisense and type-safety.

Once the provider is architected, proceed to [generators/hooks.md](../generators/hooks.md) to build the data-fetching and mutation layers.
